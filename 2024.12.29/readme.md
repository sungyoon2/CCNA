케이블링
---

> <케이블링><br>
```
1. 케이블이란 : 전기를 이용하여 신호를 전달하는 선

2. 종류
  1) TP (Twisted-pair) -> 간섭을 줄임으로써 신뢰도를 높이기 위함
    (1) UTP : Unshield .. , 작업률 높음, 
    (2) STP : Shielded .. , 한번더 절연체로 감쌈으로써 데이터 간섭을 더 줄임으로 신뢰도가 더 높아짐, 그러나 비용이 상

3. 케이블의 배열 형태
  1) T568A : 초띠/초/주띠/파/파띠/주/갈띠/갈
  2) T568B : 주띠/주/초띠/파/파띠/초/갈띠/갈

4. 케이블 종류
  1) Cross-over Cable : 같은 종류의 장치는 송수신부가 동일한 위치이므로 양쪽이 똑같은 배열이면 송수신이 이루어지지 못함
                        -> 시작 배열과 끝 배열이 다른 Cross-over cable을 사용함
      (1) 같은 type장치의 예(송/수신 규격이 동일)
          : PC-PC, Router-Router, Hub-Hub, Switch-Switch, PC-Router, Switch-Hub
  2) Direct Cable : 다른 타입의 장치일 경우 송수신부 위치가 반대이므로 굳이 꼬지않고 일자로 가도 송수신이 이루어짐
                    -> 시작 배열과 끝 배열이 동일안 Direct Cable을 사용함
      (2) 다른 Type 장치의 예 -> 위를 제외한 다른 장치간 연결
  3) Straight-though Cable(롤오버 케이블) : 장치 set up시 사용, 콘솔 PORT와 연결하기 위해 사용
                                          -> 시작배열과 끝 배열이 뒤집
```

> <케이블링 실습><br>
```
1. Cisco Packet Tracer를 이용한 케이블링 실습
2. 케이블링시 주의사항
  - L2 Switch -> 연결하자마자 되는 것이 아님 => SDT(스패닝 트리)를 이용한 통신가능 여부를 파악후 연결을 시킴
  - Routher -> 같은 타입의 장치임에도 연결불가가 뜨는 이유 => 새롭게 연결했다해서 활성화되면 기존에 연결된 내트워크에 영향을 주기때문에 세부 설정에 들어가 open을 해주어야 함

3. 통신망 개수 세기
  1) 하나의 통신망 => 네트워크 IP 1개가 필요
  2) L3미만의 장치(Router미만)의 장치로 연결시 -> 어떤 연결이든 통신망 1개 즉, 네트워크 IP도 1개만 필요함
      => L3이상의 장치들은 이미 Router기능을 내장하고 있기에 사용신 통신망개수가 늘어남
          * Router-> GW장치
  3) 각기 다른 내부통신망을 Switch로 연결시 -> 2개의 Router를 사용하는 1개의 통신망으로 취급
                                              => 통신망 하나가 사라져 같은 NW로 취급되어 하나의 IP주소가 필요함
```

IP 
---
> <IP 개요><br>
```
1. IP - 3계층 장비

2. 1 Byte = 8 Bit , Bit-> 변환된 2진 데이터 1개를 담는 단위

3. IP주소의 표현 : 32Bit로 구성, 8bit씩 4개

4. IP주소의 종류
  1) Network IP : 통신망을 구별하기 위한 IP-> 서브넷 마스크가 '0'인부분은 0으로 치며, 구분되는 IP주소 중 처음주소를 NetWork IP로 정의
  2) Host IP  : 통신망내의 기기내를 구별하기 위한 IP
  * 이를 결정(지정)하는 것 -> Subnet Mask 
                              (1) 미리 약속됨
                              (2) 1 : 네트워크 IP영역(Prefixed) -> 고정됨
                                  0 : 호스트 IP영역(Unprefixed) -> 가변
                              (3) 호스트의 범위에서 가장 마지막 숫자는 쓰지 않음 => 브로드 캐스트용
                              (4) 즉, 호스트의 범위는 0과 마지막 수를 제외한 것 -> 총 수에 -2를 해야함

5. 클래스풀 : IP분배방식, IANA에서 만듬      
  1) 정해놓은 Subnet Mask를 이용해서 서로 다른 주소영역으로 나눔
  2) 클래스          서브넷마스크             식별용 상위비트         주소범위                      네트워크망 크기(호스트 IP 범위가 많이 비어있면 커짐)
       A            255.0.0.0(/8)              0XXXXXXX        0.0.0.0 ~ 127.255.255.255              24비트가 빔
       B            255.0.0.0(/16)             10XXXXXX        128.0.0.0 ~ 191.255.255.255            16비트가 빔
       C            255.0.0.0(/24)             110XXXXX        192.0.0.0 ~ 223.255.255.255            8비트가 빔
       D            멀티캐스팅용                1110XXXX        224.0.0.0 ~ 239.255.255.255
       E            실험용,장래대비(Mask이용x)   1111XXXX        240.0.0.0 ~ 255.255.255.255
    cf) 클래스리스 : 규격화된 구분없이 비트단위별로 ip 주소 범위를 가변적으로 구현

6. 통신 방법
  1) Unicast : 1대1 연결
  2) Broadcast : 모든 Host에게 뿌림 -> 보안의 문제가 발생(취약점 발생), 트래픽낭비, 회선에 부담이 높음
  3) Multicast : 특정 Host를 그룹지어서 연결(전송)
```
